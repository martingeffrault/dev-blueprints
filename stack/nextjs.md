## Philosophy (2025)

Next.js 15 solidifies the App Router as the only recommended approach. Pages Router is legacy.

**Key shifts:**
- App Router is THE way — no new projects should use Pages Router
- Server Components everywhere, 'use client' only when needed
- Server Actions for mutations (no more API routes for simple mutations)
- Partial Prerendering (PPR) for optimal performance
- Built-in caching (fetch cache, router cache, full route cache)
- Turbopack is now stable for development
